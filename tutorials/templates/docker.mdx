## What You'll Build

A local AgentOS development environment with Docker Compose, including:
- AgentOS FastAPI server
- PostgreSQL database with pgvector

This setup is ideal for development, testing, and verifying your agents work correctly before deploying to production.

## Quick Start

<Steps>
<Step title="Install tools">

Install [uv](https://docs.astral.sh/uv/) (Python package manager) and Agno:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
uv pip install -U "agno[infra]"
```

</Step>
<Step title="Create the project">

```bash
ag infra create
```

Select "Docker" when prompted. This creates a new `agent-infra-docker` directory.

<Tip>
Alternatively, clone directly: `git clone https://github.com/agno-agi/agent-infra-docker.git`
</Tip>

</Step>
<Step title="Set your API key">

```bash
export OPENAI_API_KEY=sk-***
```

<Tip>
Agno works with any model provider. Update the agents in your project to use Claude, Gemini, or other models.
</Tip>

</Step>
<Step title="Start the containers">

```bash
ag infra up
```

Wait a few seconds for containers to start.

</Step>
<Step title="Verify it works">

Open http://localhost:8000/docs in your browser. You should see the FastAPI documentation for your AgentOS instance:

<Frame>
  <img src="/images/agentos-endpoints.png" alt="AgentOS FastAPI endpoints" />
</Frame>

</Step>
<Step title="Connect to AgentOS UI">

Connect your local AgentOS to the control plane for testing and monitoring:

1. Open [os.agno.com](https://os.agno.com)
2. Click "Connect OS" and select "Local"
3. Enter your endpoint URL (http://localhost:8000)

<Frame>
  <img src="/images/agent-os-connection-dialog.png" alt="AgentOS connection dialog" />
</Frame>

</Step>
</Steps>

## Development Workflow

### View logs

```bash
docker compose logs -f
```

### Make changes

Edit your agents, teams, or workflows in the codebase. Changes are reflected automatically with hot reload.

### Restart containers

```bash
ag infra restart
```

### Stop containers

```bash
ag infra down
```

## Project Structure

```
agent-infra-docker/
├── agents/              # Your agents
├── teams/               # Your teams
├── workflows/           # Your workflows
├── app/main.py          # AgentOS entry point
├── docker-compose.yml   # Docker configuration
├── Dockerfile           # Container build
└── pyproject.toml       # Python dependencies
```

## Troubleshooting

### Port already in use

If port 8000 is in use, modify `docker-compose.yml`:

```yaml
ports:
  - "8080:8000"  # Change 8000 to 8080
```

### Database connection issues

Ensure PostgreSQL container is running:

```bash
docker compose ps
```

If the database isn't ready, wait a few seconds and try again.
