---
title: Set Session Names
description: Learn how to set custom names for your sessions
---

Session names are human-readable labels that help you identify and organize conversations. Instead of tracking sessions by UUID, you can assign meaningful names like "Product Launch Discussion" or "Customer Support - Issue #123".

## Why Use Session Names?

- **Better UX**: Show meaningful names in conversation history UIs
- **Organization**: Group and search sessions by name
- **Context**: Quickly understand what a session is about
- **Integration**: Map sessions to external systems (tickets, tasks, etc.)

## Basic Example

<Tabs>
  <Tab title="Agent">
```python
from agno.agent import Agent
from agno.db.sqlite import SqliteDb
from agno.models.openai import OpenAIChat

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    db=SqliteDb(db_file="tmp/agent.db"),
)

# Have a conversation
agent.run("I need help planning a product launch")
agent.run("We're targeting enterprise customers")

# Set a descriptive name
agent.set_session_name(session_name="Product Launch Planning")

# Retrieve the name
session_name = agent.get_session_name()
print(f"Session: {session_name}")  # Output: Session: Product Launch Planning
```
  </Tab>

## Setting Names for Specific Sessions

You can manage multiple sessions and set names for each:

<Tabs>
  <Tab title="Agent">
```python
from agno.agent import Agent
from agno.db.postgres import PostgresDb
from agno.models.openai import OpenAIChat

db = PostgresDb(
    host="localhost",
    port=5432,
    user="agno",
    password="agno",
    database="agno",
)

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    db=db,
)

# Session 1: Bug investigation
agent.run("Help me debug this error", session_id="session_001")
agent.set_session_name(
    session_id="session_001",
    session_name="Bug Investigation - Login Issue"
)

# Session 2: Feature planning
agent.run("Let's plan the new feature", session_id="session_002")
agent.set_session_name(
    session_id="session_002",
    session_name="Feature Planning - Dark Mode"
)

# Retrieve names
name1 = agent.get_session_name(session_id="session_001")
name2 = agent.get_session_name(session_id="session_002")

print(f"Session 1: {name1}")  # Bug Investigation - Login Issue
print(f"Session 2: {name2}")  # Feature Planning - Dark Mode
```
  </Tab>

## Renaming Sessions

You can update session names at any time:

<Tabs>
  <Tab title="Agent">
```python
agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    db=SqliteDb(db_file="tmp/agent.db"),
)

# Start with one name
agent.run("Initial message", session_id="my_session")
agent.set_session_name(session_id="my_session", session_name="Initial Discussion")

# Later, update the name based on how the conversation evolved
agent.run("Actually, let's focus on the API design", session_id="my_session")
agent.set_session_name(session_id="my_session", session_name="API Design Discussion")

print(agent.get_session_name(session_id="my_session"))
# Output: API Design Discussion
```
  </Tab>

  <Tab title="Team">

## Integration with External Systems

Session names are perfect for integrating with ticketing systems, task managers, or CRMs:

<Tabs>
  <Tab title="Agent">
```python
from agno.agent import Agent
from agno.db.postgres import PostgresDb
from agno.models.openai import OpenAIChat

db = PostgresDb(
    host="localhost",
    port=5432,
    user="agno",
    password="agno",
    database="agno",
)

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    db=db,
)

# Customer support example
ticket_id = "SUPPORT-12345"
customer_name = "Acme Corp"

# Start session with ticket context
response = agent.run(
    "Customer is having login issues",
    session_id=ticket_id,
    metadata={"ticket_id": ticket_id, "customer": customer_name}
)

# Set name that matches your ticket system
agent.set_session_name(
    session_id=ticket_id,
    session_name=f"Support - {customer_name} - Login Issue"
)

# Later, retrieve by ticket ID
session_name = agent.get_session_name(session_id=ticket_id)
print(f"Working on: {session_name}")
```
  </Tab>

## Async Support

Session naming works with async operations:

<Tabs>
  <Tab title="Agent">
```python
import asyncio
from agno.agent import Agent
from agno.db.postgres import PostgresDb
from agno.models.openai import OpenAIChat

async def manage_session():
    db = PostgresDb(
        host="localhost",
        port=5432,
        user="agno",
        password="agno",
        database="agno",
    )
    
    agent = Agent(
        model=OpenAIChat(id="gpt-4o"),
        db=db,
    )
    
    # Run async
    await agent.arun("Start conversation", session_id="async_session")
    
    # Set name async
    await agent.aset_session_name(
        session_id="async_session",
        session_name="Async Discussion"
    )
    
    # Get name async
    name = await agent.aget_session_name(session_id="async_session")
    print(f"Session: {name}")

asyncio.run(manage_session())
```
