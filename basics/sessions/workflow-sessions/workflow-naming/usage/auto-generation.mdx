## Basic Auto-Generation

Let Agno generate a timestamp-based name:

```python
from agno.workflow import Workflow
from agno.db.sqlite import SqliteDb

workflow = Workflow(
    name="Research Pipeline",
    db=SqliteDb(db_file="workflows.db"),
    steps=[...],
)

workflow.run(input="Research topic")
workflow.set_session_name(autogenerate=True)
# Result: "Workflow Session - 2024-11-19 14:30"
```

## With Description

When you provide a workflow description, it's included in the generated name:

```python
workflow = Workflow(
    name="Research Pipeline",
    description="Automated research and analysis pipeline",
    db=SqliteDb(db_file="workflows.db"),
    steps=[...],
)

workflow.run(input="Research topic")
workflow.set_session_name(autogenerate=True)
# Result: "Automated research and analysis pipel - 2024-11-19 14:30"
```

<Note>
Descriptions longer than 40 characters are truncated to keep names manageable.
</Note>

## Format Details

Auto-generated names follow these formats:

**Without description:**
```
"Workflow Session - YYYY-MM-DD HH:MM"
```

**With description:**
```
"{description_truncated_40} - YYYY-MM-DD HH:MM"
```

## Async Version

```python
await workflow.aset_session_name(
    autogenerate=True,
    session_id="async-session-2"
)
```

## When to Use Auto-Generation

Good for:
- Quick prototyping
- Consistent timestamp-based tracking
- When you don't need custom names
- Logging and debugging

Better to use manual naming when:
- You need specific identifiers
- Working with business contexts
- Names need to follow a pattern
