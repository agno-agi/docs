An intelligent invoice processing agent that extracts structured data from invoice documents using vision capabilities. Handles PDFs and images, adapts to different layouts without templates, and validates extracted data.

## What It Does

- Processes invoices in various formats (PDF, PNG, JPG)
- Extracts structured data using vision capabilities
- Handles different invoice layouts without templates
- Validates totals against line item sums
- Returns confidence scores and data quality warnings

## Prerequisites

- Python 3.12+
- Anthropic API key (for Claude with vision)
- poppler (system dependency for PDF to image conversion)

## Agent Configuration

```python
invoice_agent = Agent(
    name="Invoice Analyst",
    model=Claude(id="claude-sonnet-4-5"),
    system_message=SYSTEM_MESSAGE,
    output_schema=InvoiceData,
    tools=[
        ReasoningTools(add_instructions=True),
    ],
    markdown=True,
)
```

| Parameter | Purpose |
|-----------|---------|
| `model` | Claude Sonnet with vision capabilities |
| `output_schema` | Pydantic model for structured invoice data |
| `ReasoningTools` | Plan extraction approach and validate data |

The agent uses Claude's native vision capabilities. No additional vision tools are needed.

## How It Works

### Extraction Workflow

```
1. Load invoice document
2. Convert to image(s) if PDF
3. Send to Claude with vision capabilities
4. Extract fields using visual understanding
5. Parse line items table
6. Validate totals and calculations
7. Return structured data with confidence score
```

### Output Schema

```python
class InvoiceData(BaseModel):
    invoice_number: str
    invoice_date: date
    due_date: date | None
    vendor: Vendor
    line_items: list[LineItem]
    subtotal: Decimal
    tax_amount: Decimal | None
    total_amount: Decimal
    currency: str
    confidence_score: float
    warnings: list[str]
```

### Validation Rules

| Check | Formula |
|-------|---------|
| Line item math | `quantity × unit_price = amount` |
| Subtotal | Sum of line items ≈ subtotal |
| Total | `subtotal + tax - discount + shipping ≈ total` |

## Troubleshooting

<AccordionGroup>
  <Accordion title="PDF conversion fails">
    Ensure poppler is installed:
    ```bash
    # macOS
    brew install poppler

    # Ubuntu
    apt-get install poppler-utils
    ```
  </Accordion>
  <Accordion title="Low confidence on scanned invoices">
    Scanned documents may have poor image quality, skewed alignment, or background noise. Try improving the scan quality.
  </Accordion>
  <Accordion title="Math validation warnings">
    Common causes: rounding differences, hidden fees, or multi-page invoices with partial totals. Review warnings and verify manually if needed.
  </Accordion>
</AccordionGroup>
